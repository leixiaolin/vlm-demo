# 室内办公图片抓拍风险行为智能分析预研包

本仓库用于验证“云端多模态大模型 + 图片描述/结构化风险判断”在室内办公重点场所人员风险行为分析中的可行性。当前交付不是完整产品，而是一套可复现实验资产：技术报告、接口草案、风险标签、提示词、评测表模板和评估脚本。

## 目录

- `docs/technical_research_report.md`：技术预研报告，包含架构、能力边界、模型路线、验证指标和结论口径。
- `docs/interface_contract.md`：图片分析接口草案、字段定义、阈值策略和人工复核规则。
- `docs/poc_next_steps.md`：若预研通过，下一阶段轻量 PoC 的建设建议。
- `docs/data_collection.md`：公开室内办公图片采集脚本说明。
- `docs/image_description_test.md`：批量图片描述测试流程说明。
- `docs/ovis25_local_eval.md`：本地部署 Ovis2.5-2B 并测试 data 图片的流程说明。
- `docs/gar1b_local_eval.md`：本地部署 GAR-1B 并测试 data 图片的流程说明。

- `prompts/`：零样本、少样本、强结构化三类实验提示词。
- `data/`：评测集清单、人工标注表、实验记录表与样例 JSONL。
- `scripts/collect_office_images.py`：从 Openverse/Wikimedia Commons 公开媒体 API 收集到 150-300 张室内办公有人活动图片，并生成来源与评测清单。
- `scripts/run_image_description_test.py`：批量调用视觉模型生成图片描述 JSONL，支持 mock 本地验证和 OpenAI Responses API。
- `scripts/run_ovis25_local_eval.py`：本地加载 Ovis2.5-2B，对 data 图片执行安全风险检测并记录性能。
- `scripts/run_gar1b_local_eval.py`：本地加载 GAR-1B，对 data 图片执行安全风险检测并记录性能。
- `scripts/summarize_ovis25_results.py`：汇总 Ovis2.5-2B 本地测试结果。
- `scripts/evaluate_results.py`：本地评测脚本，计算准确率、误报率、漏报率、结构化合规率等指标。
- `scripts/build_research_workbook.mjs`：根据 CSV/JSONL/评测结果生成 Excel 预研工作簿。
- `outputs/`：评测脚本输出目录。

## 推荐实验流程

1. 收集 150-300 张脱敏、公开或合成的室内办公图片，覆盖正常样本、明确风险样本和困难负样本。
2. 按 `data/evaluation_manifest.csv` 建立样本清单，按 `data/experiment_record.csv` 记录每轮模型输出。
3. 先跑 `prompts/zero_shot.md`，再跑 `prompts/few_shot.md`，最后跑 `prompts/structured_schema.md`。
4. 将模型输出保存为 JSONL，每行一个图片分析结果，字段遵循 `schemas/image_risk_analysis_response.schema.json`。
5. 使用 `scripts/evaluate_results.py` 统计结果，并在报告中记录典型失败案例。

## 预研边界

- 只验证图片抓拍，不验证实时流、长视频时序、身份识别和跨摄像头追踪。
- 只上传已授权、脱敏、模拟或公开样本，真实办公图片不默认出域。
- 模型结论只作为风险提示，所有中高风险和低置信关键风险默认进入人工复核。
- 单帧无法确认的行为必须输出“疑似”或“不足以判断”，不能臆测身份、动机、情绪或法律结论。


可以的，这个方法在工程上完全可行，本质上是用**“关键帧采样 + 静态图像理解 + 跨帧信息融合”**来近似视频理解。我先直接回答你的问题，再说明它的能力边界。

**一、为什么这个方法可行**

现在的多模态大模型（如GPT-4o、Gemini等）对单张图片的语义描述能力已经很强，能识别物体、人物、场景、文字、表情、动作状态甚至空间关系。

流程上你只需：

- 对视频按时间或按场景变化抽帧
- 每帧（或选关键帧）调用大模型，生成自然语言描述
- 把多帧描述拼接成一篇文本，再让大模型总结时序变化和全局内容

这个**“视频→图片序列→文本序列→总结文本”**的管道，在没有原生视频理解模型或做批处理、日志分析时很实用。

**二、效果到底好不好，取决于这几个因素**

- **采样密度**：抽帧太少，会漏掉关键动作和突变。对变化快的视频（如体育、舞蹈、第一人称操作），需要高帧率采样。
- **描述质量**：每帧描述越详细越好，最好要求模型输出“动作、人物状态、界面变化、场景转换”等结构化信息。
- **时序信息保留**：这是最大弱点。静态描述会丢失连续运动和时序因果。一定要在每一帧描述里带上时间戳，并在总结阶段显式要求模型提取趋势和顺序。

**三、能处理到什么程度**

**较擅长的场景：**

- 幻灯片/教程视频（画面以图文为主）
- 监控式的场景变化检测（谁出现、物体是否被移动）
- 界面操作录屏（每一步按钮和状态变化）
- 缓慢变化的环境与活动记录

**明显不足的场景：**

- 需要理解连续动作的（体育技巧、舞蹈、手术操作）
- 对话密集、依赖唇语和微表情的
- 快速运镜、剪辑密集的短视频
- 需要精确时间定位的事件

**四、怎么增强效果（不需要换模型就能提升）**

有几个很有效的做法：

- **让每帧描述带上时间戳**：“[00:05] 一个男人在厨房，右手拿起杯子；[00:07] 杯子接近嘴边……”
- **要求总结时专门提炼变化**：提示词里加入“请比较相邻时间段的描述，列出发生的变化及其顺序”
- **先筛选关键帧再描述**：用简单的帧间差异（像素变化或特征相似度）去重，可以减少冗余、降低成本并突出变化点
- **结合音频转写**：把语音转文字与帧描述交织在一起，理解准确度会大幅提升

**总结一下**：这个方法在实际工程中经常被用作轻量级视频理解的起点，对内容相对静态、结构化强的视频效果不错，还能批量处理。但它不能替代真正的视频模型去理解连续性动作和细微时序因果。如果你想用它来分析某类具体视频，我可以帮你设计抽帧策略和提示词模板。



针对“内部办公场所窃密与数据泄露风险”这类主题的视频，通常属于**安全培训、风险演示、场景复盘或渗透测试记录**。这类视频的核心特征在于：**行为动作的连续性强、风险细节多且微小、场景切换代表风险类型的变化**。

为了用“抽帧+图片描述+总结”这套方法最大程度还原视频内容，我们需要采用**基于变化检测的非均匀抽帧 + 结构化描述 + 风险点聚合总结**的策略。

以下是为你设计的完整方案。

### 一、抽帧策略设计

不能使用固定间隔抽帧，必须采用**双重策略结合**，确保不漏掉关键动作和微小细节。

**核心思路**：先用高频率抽帧捕捉动作流，再用变化检测剔除大量冗余静态帧，保留关键帧。

#### 1. 前置准备：视频预处理与元数据设置

- **分场景处理**：如果视频过长，建议先按主题（如“办公区风险”、“会议室风险”、“打印区风险”）粗略切分，以便总结时能按场景归类。
- **时间戳烙印**：在抽帧前，用脚本将视频的时间码（HH:MM:SS.mmm）硬字幕烙印在画面角落（如右下角），这对后续追溯风险动作发生的精确时刻至关重要。

#### 2. 双阶段抽帧流程

**阶段一：高频初步采样**

- **采样频率**：**每秒 2-3 帧**。
- **目的**：确保快速动作（如快速偷拍屏幕、顺走桌上文件）能被捕捉到至少2-3个阶段（准备、执行、结束）。

**阶段二：基于图像相似度的去重与关键帧提取**

- **方法**：计算相邻帧之间的结构相似性（SSIM）或直方图差异。
- **阈值设定**：
  - **高变化阈值**：用于保留场景切换、人物入场/出场等重大变化。
  - **低变化阈值**：用于保留细微但关键的动作，如手部在屏幕前的微小移动（偷拍）、抽出文件的一角。
- **输出规则**：
  1. **场景切换帧**（如从办公区切到会议室）：强制性保留，并标记为 `[场景切换]`。
  2. **动作峰值帧**：在高频采样区间内，SSI变化率最大的那一帧（动作幅度最大），保留并标记为 `[动作峰值]`。
  3. **状态起止帧**：一个连续动作开始前和结束后的稳定帧，各保留一张。例如，手伸向文件的第一帧，和文件被拿走后空桌面的第一帧。
  4. **强制抽样帧**：如果连续5秒以上都无变化，至少抽取1帧作为环境背景帧，标记为 `[静态环境]`。

### 二、图片描述提示词模板

对每一张抽出并打标的图片，我们需要生成结构化的描述，而不是一句笼统的话。描述需要成为后续分析的可靠依据。

#### 单帧描述提示词（发给多模态大模型）

```
【角色与任务】
你是一名专业的信息安全风险评估分析师。请仔细分析这张来自内部办公场所监控或演示视频的关键帧图片，生成一份结构化的风险观察报告。

【画面元数据】
- 画面时间戳：[请在此处填入图片上烙印的时间，如 00:15:23.200]
- 关键帧类型：[请填入类型，如：场景切换 / 动作峰值 / 状态起止 / 静态环境]

【结构化描述要求】
请严格按以下格式输出，不要遗漏任何部分：

1. **场景与环境**：
   - 地点类型：开放工位/独立办公室/会议室/打印区/走廊/茶水间等。
   - 环境特征：玻璃隔断、白板、共享打印机、临窗工位等潜在风险载体。

2. **人员与行为**：
   - 图中人物数量、各自大致位置与身份特征（如员工、访客、保洁）。
   - 每个人的具体动作：例如“左手持手机在屏幕上方做拍照状”、“弯腰查看他人桌面文件”、“尾随通过门禁”。
   - **关键风险行为判定**：根据图像内容，显式判断该行为属于哪类风险（如：肩窥、偷拍屏幕、文件翻看、尾随、私接设备等），并给出判定依据。

3. **敏感信息暴露点**：
   - 肉眼可见的屏幕内容（概括主题，如“显示着代码/财务表格/组织架构图”）。
   - 暴露的物理文件、便签、白板字迹、工牌等。
   - 如果有水印、加密等保护措施，也请指出。

4. **环境异常点**：
   - 不合规的情况：如屏幕未锁屏且无人、敏感文件遗留在空位、碎纸机旁有未粉碎文件、临时访客无工牌等。
```

### 三、多帧描述总结提示词模板

当一段视频的所有关键帧被描述完毕后，我们需要将这些结构化文本“串联”成一个完整的故事，并提炼出整体风险。

#### 多帧聚合总结提示词（发给文本大模型）

```
【角色与任务】
你是一名资深的信息安全顾问。现在，你将收到一段内部办公场所安全演示视频的连续关键帧描述列表。请基于这些时序描述，重构视频中的事件流，并进行全面的风险分析总结。

【输入数据格式】
你将收到一份按时间顺序排列的JSON列表，每条记录包含：
- timestamp: 画面时间戳
- frame_type: 关键帧类型
- description: 上述【结构化描述要求】生成的四部分内容。

【总结生成要求】
请通读所有帧描述，完成以下分析，并生成一份《视频安全分析报告》：

1. **视频核心事件时间线**：
   - 用`[时间戳] 事件描述`的格式，梳理出视频中发生的所有风险事件流。
   - 将帧与帧之间连续的、属于同一事件的行为合并为一条故事线。例如，不要逐帧描述“站在工位旁”，而是总结为“00:05:10 - 00:05:45，一名无访客工牌人员在过道徘徊，并两次低头查看Y工位桌面上的文件”。

2. **发现的风险点汇总**：
   - 将所有风险行为归纳为几大类（如：视觉窃密、物理介质泄露、人员违规等）。
   - 为每类风险列出在本视频中观察到的具体案例作为证据。

3. **环境与流程脆弱性分析**：
   - 分析视频中暴露的物理环境缺陷（如：玻璃墙未贴膜、打印机无刷卡认证）。
   - 分析员工或流程的薄弱环节（如：离席未锁屏、让陌生人尾随进门、文件随意放置）。

4. **整改建议**：
   - 针对视频中发现的每一项具体风险，给出可落地的、分层的整改建议（技术、物理、管理三个层面）。
```

### 四、实战优化建议

1. **预定义风险标签集**：在单帧描述时，你可以给模型一个预设的风险标签列表供其选择，如 `[非法拍照] [肩窥] [桌面遗留] [尾随] [私接设备] [白板未擦] [屏幕未锁]` 等，这能让描述更标准化，易于后续统计。
2. **处理连续动作**：对于快速偷拍这类一秒内完成的动作，前后3-5帧会被连续描述为“偷拍”。总结时的提示词明确要求“将连续、同一人物的同一风险行为合并为一次事件”，避免把一个动作误报成多次。
3. **结合音频的特别处理**：如果视频有音频（如培训讲解），可先用ASR转写，并按时间戳与帧描述交织。总结时，大模型就能将“视频动作”与“讲师讲解”对应起来，理解视频的叙事意图（例如：知道这是一个演示正确行为的，还是演示反面案例的）。

这套策略专门适配于你关注的这类高细节、高时序关联的风险主题视频，能最大限度发挥“静态图+大模型”方案的能力。需要的话，我可以针对某个具体风险场景（比如防尾随、防偷拍）再做更定制化的调整。
