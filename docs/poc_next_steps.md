# 后续轻量 PoC 建议

如果预研达到通过或有条件通过标准，建议进入 4-6 周轻量 PoC。

## PoC 目标

- 支持图片上传、批量分析、结构化风险结果展示。
- 支持人工复核、纠错、导出实验记录。
- 支持阈值配置、风险标签开关和模型对比。
- 形成可演示、可复验、可扩展到视频流的最小系统。

## 功能范围

第一阶段应包含：

- 图片上传与样本管理。
- 单张/批量模型调用。
- 风险列表、证据文本、置信度、严重度展示。
- 人工复核结果记录。
- 实验指标面板：准确率、误报率、漏报率、复核命中率、平均响应时间、单张成本。

第一阶段暂不包含：

- 摄像头实时接入。
- 人脸识别或身份绑定。
- 自动处罚或工单闭环。
- 大规模模型微调。

## 技术建议

- 后端：提供统一 `analyze_image` 适配层，屏蔽不同 VLM API 差异。
- 数据：使用对象存储保存脱敏图片，关系库保存标注、输出和复核结果。
- 前端：以评测和复核为主，避免做成监控大屏。
- 运维：记录模型版本、提示词版本、schema 版本、调用耗时和成本。

## 里程碑

| 周期 | 目标 |
| --- | --- |
| 第 1 周 | 完成样本集、标注规范和接口适配 |
| 第 2 周 | 完成单张分析和结构化结果入库 |
| 第 3 周 | 完成批量评测、指标统计和失败案例库 |
| 第 4 周 | 完成人工复核界面和阈值配置 |
| 第 5-6 周 | 做现场样本试运行、调优并输出 PoC 总结 |
